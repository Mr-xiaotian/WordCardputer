# ModeStats.ino

## 概述
`ModeStats.ino` 实现了**学习统计与报表模式**。它对当前内存中已加载的词库数据进行数学分析，并在屏幕上以多页报表的形式向用户展示学习进度的宏观情况。该模块是评估用户对当前词书掌握程度的核心。

## 核心内容
- **数据统计算法 (`computeStatsFromWords`)**：
  - **总数与分布**：遍历当前加载的 `words` 数组，获取单词总数（`statsTotal`），并将每个单词的 `score`（熟练度得分，1-5）分桶统计，记录到 `statsCounts` 数组中。
  - **平均分计算**：累加所有分数值求取平均值（`statsAvg`），反映整体熟练度。
  - **中位数计算**：为了排除极值干扰，通过对分数数组进行排序，提取出中位数（`statsMedian`），这能更真实地反映“大部分单词处于什么水平”。
  - **综合评级生成**：根据平均分、中位数以及满分词汇的占比，生成一个文字形式的 `statsLevel`（例如：“掌握良好”、“急需复习”、“初学阶段”等），给予用户直观的反馈。
- **分页渲染展示 (`drawStats`, `loopStatsMode`)**：
  - **多页设计**：由于 M5Cardputer 屏幕较小，数据被拆分为三页展示，通过 `statsPage` 变量控制当前页。
    - **第 1 页（概览）**：显示文件名、总词数、平均分、中位数和综合评价。
    - **第 2 页（分布上）**：利用 `UtilsMenu.ino` 的 `drawSimpleTable`，以表格形式展示得分 1 到 3（不熟练阶段）的单词数量及其在总数中的百分比。
    - **第 3 页（分布下）**：同样以表格展示得分 4 到 5（熟练阶段）的单词分布情况。
  - **交互控制**：用户可以通过方向键（`;/.`）在三页报表之间来回翻页，按 `ESC` 键退出统计并返回之前的模式（或主菜单）。

## 关联模块
- **数据来源**：直接访问全局的 `words` 数组（在 `WordCardputer.ino` 声明，由 `UtilsData.ino` 加载）。
- **Web API 复用**：其核心统计算法 `computeStatsFromWords()` 也会被 `UtilsWebServer.ino` 调用，用于向 Web 前端提供 JSON 格式的统计数据接口。
- **UI 渲染**：依赖 `UtilsMenu.ino` 提供的表格渲染能力。
