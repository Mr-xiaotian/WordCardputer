# ModeDictation.ino

## 概述
`ModeDictation.ino` 实现了**听写测验模式**，这是一个高强度的输出练习功能。它播放单词发音并显示中文释义，要求用户通过键盘拼写出对应的单词。为了支持日语输入，该模块内置了一个轻量级的罗马音到假名的输入法引擎。

## 核心内容
- **测试流程控制 (`initDictationMode`, `loopDictationMode`)**：
  - **初始化**：生成一个打乱的索引数组（`dictOrder`），按照随机顺序对当前词库进行测试。目前英语的听写功能暂未开放（会显示提示并退出）。
  - **测验循环**：
    - 播放当前单词的音频。
    - 监听用户键盘输入，动态渲染输入框内容。
    - 按 `Enter` 键提交答案，程序会将输入内容与正确答案（`w.jp` 或 `w.en`）进行严格比对。
    - 记录比对结果：如果错误，将单词信息保存到 `dictErrors` 列表中。
    - 自动切入下一个单词。
  - **结算与复盘 (`drawDictationResult`, `loopDictationResult`)**：
    - 听写结束（所有词测完或用户中途交卷）后，展示成绩单（正确率、耗时等）。
    - 允许用户进入“错题回顾”模式，逐个查看拼错的单词。
    - 提供选项将错题导出为独立的复习文件（调用 `saveDictationMistakesAsWordList`）。
- **内置日文输入法 (`processRomajiInput`)**：
  - **Level 1 罗马音转换**：这是本模块最具技术含量的部分。它维护了一个 `romajiBuffer`（当前按键序列），通过硬编码的查表逻辑（或规则前缀匹配），将类似 "ka"、"shi"、"tsu" 的英文字符序列实时转换为对应的平假名。
  - **候选与提交**：当输入的字母不足以唯一确定一个假名时，会显示为“候选（candidate）”状态；一旦匹配成功，自动提交到 `commitText`（答案栏）中，并清空缓冲区等待下一个音节。
  - **删除逻辑**：按 `Backspace`（退格键）时，优先删除未完成的罗马音缓冲区；若缓冲区为空，则删除最后一个已提交的假名字符。

## 关联模块
- **音频提示**：依赖 `UtilsAudio.ino` 播放测试题发音以及正误提示音。
- **数据回写**：依赖 `UtilsData.ino` 创建包含时间戳的错题本 JSON 文件。
