# ModeListen.ino

## 概述
`ModeListen.ino` 实现了**听读模式**（又称“磨耳朵”模式）。这是一种免提（Hands-free）的学习方式，系统会自动挑选单词，在屏幕上展示内容的同时循环播放其音频，适合在做其他事情时作为背景音学习。

## 核心内容
- **语言限制守护**：
  - 目前设计上主要针对日语设计，在初始化（`initListenMode`）时会检查当前语言，如果是英语（`LANG_EN`），会显示“英语听读暂未支持”并等待用户按 `ESC` 退出。
- **自动播放状态机 (`loopListenMode`)**：
  - 基于时间戳（`millis()`）实现的非阻塞延迟控制，包含以下状态流转：
    1. **选词**：通过 `UtilsData.ino` 中的 `pickWeightedRandom()` 抽取一个不熟练的单词。
    2. **播放**：调用 `playAudioForWord` 播放发音。
    3. **等待**：记录当前时间，并设置 `listenNextActionTime = millis() + listenRepeatInterval`。
    4. **重复**：当时间到达时，如果当前单词播放次数不足 3 次（由 `listenPlayCount` 控制），则重新播放；否则切入下一个单词的延迟阶段。
- **界面渲染 (`drawListenWord`)**：
  - 动态显示当前播放的单词。
  - 使用大字号显示假名（日语），并在下方显示音调和中文释义。
  - 在屏幕底部动态显示进度条：例如 `[播放 2/3]`，让用户明确当前的播放状态。
- **快捷交互**：
  - 在自动播放过程中，用户仍可以使用键盘的上下键快速调整系统音量，屏幕左下角会即时显示音量变化（如 `Vol: 80`）。
  - 按 `ESC` 键可随时跳出自动循环并打开主菜单。

## 关联模块
- **数据**：依赖 `UtilsData.ino` 提供加权选词算法。
- **音频**：重度依赖 `UtilsAudio.ino` 提供的音频播放和静音控制能力。
